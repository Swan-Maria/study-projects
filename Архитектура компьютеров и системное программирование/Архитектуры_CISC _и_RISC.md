# Архитектуры CISC и RISC

## 1. Основные понятия
ISA (Instruction Set Architecture) — набор команд, их форматы, способы адресации и правила работы процессора.  
Существует два основных подхода: **CISC** и **RISC**.

---

## 2. CISC (Complex Instruction Set Computer)
### Характеристики
- Большой и сложный набор инструкций.
- Команды разной длины.
- Много режимов адресации.
- Сложные инструкции: одна команда выполняет несколько операций.
- Использование микрокода.

### Плюсы
- Компактный машинный код.
- Меньше команд для сложных операций.
- Легче компиляторам.

### Минусы
- Сложная реализация процессора.
- Трудная конвейеризация.
- Ниже предсказуемость выполнения.

### Примеры
x86, x86-64, VAX, Motorola 68000.

---

## 3. RISC (Reduced Instruction Set Computer)
### Характеристики
- Малый, простой набор инструкций.
- Фиксированная длина команд.
- Load/Store: работа с памятью только через регистры.
- Много универсальных регистров.
- Простые режимы адресации.

### Плюсы
- Быстрая конвейеризация.
- Простота аппаратуры.
- Высокая производительность на такт.
- Низкое энергопотребление.

### Минусы
- Более длинные программы.
- Высокие требования к компиляторам.

### Примеры
ARM, ARM64, MIPS, RISC-V, PowerPC, SPARC.

---

## 4. Краткое сравнение
| Параметр             | CISC                         | RISC                         |
|----------------------|------------------------------|------------------------------|
| Количество команд    | большое                      | маленькое                    |
| Длина команд         | переменная                   | фиксированная                |
| Доступ к памяти      | разрешён многим инструкциям  | только Load/Store            |
| CPI                  | >1                           | ≈1                           |
| Конвейеризация       | сложная                      | эффективная                  |
| Сложность CPU        | высокая                      | низкая                       |
| Плотность кода       | высокая                      | ниже                         |

---

## 5. Итог
- **CISC** — сложные инструкции, компактный код, тяжёлая аппаратная реализация.  
- **RISC** — простые инструкции, высокая производительность и предсказуемость.  
Современные процессоры часто комбинируют оба подхода.
